Go Engine Naming System

One word, one meaning. Five words cover everything.
point
one intersection
on the board
board
the stone map
(all points together)
state
board + game context
(turns, ko, passes)
action
what the engine
does this turn
move
what a human reads
in the game record

How They Nest

state (GameState object) ├── boards ─────── (B, N2) int8 stone at every point ├── to_play ───── (B,) int8 whose turn ├── pass_count ── (B,) int8 consecutive passes └── zobrist_hash (B, 2) int32 current + previous hash

Code Examples

What's at this point?
# point_id is the address # boards is the container stone = engine.boards[b, point_id]
Which points are legal?
# returns (B, N2) bool # one bool per point, all games legal = engine.legal_points()
Place a stone at a point
# direct write to the board engine.boards[b, point_id] = Stone.BLACK
Apply a chosen action
# action_id in [0, N2] # N2 = pass, 0..N2-1 = place engine.state_transition(action_ids)
Show board to a human
# reshape for 2D display # no adapter API needed boards_2d = engine.boards.view(B, H, H)
Write a game record
# human-facing JSON entry { "move_number": ply + 1, "action_id": a, "row": r, "col": c }

Naming by Layer

Engine
state board point action
legal_points()   state_transition(action_ids)
1D canonical. No _flat suffix needed.
Agents / Search
action_id action_ids point_id
select_actions(engine) returns (B,) action IDs
Agents consume legal_points, produce action_ids.
History / Log
move move_number action_id
JSON: {"move_number": 42, "action_id": 15, "row": 1, "col": 6}
Human-readable move + machine-traceable action_id.
